AIDEEN TOKEN CIRCUIT TRACE
==========================

Purpose
-------
This document is the authoritative living trace of the token lifecycle through AIDEEN.
It must be updated whenever the model path, memory path, training path, GPU layout, or debugging understanding changes.

Rule for completeness
---------------------
For every variable/function/buffer/env flag mentioned here, all transitive definitions, writes, reads, layout assumptions, and CPU/GPU crossings must be searched and understood before declaring the path stable.

0. Sequence construction
------------------------
File/function: aideen-training-lab/src/bin/assoc_recall.rs::make_ar_sequence
Role:
- Creates associative recall sequences.
- Format for pairs=1: KEY_i VAL_i FILLER*gap KEY_i QUERY VAL_i.
- Format for pairs>1: (KEY_j VAL_j)*N FILLER*gap KEY_q QUERY VAL_q.
- The final VAL token is the answer target.

Important tokens:
- KEY tokens: TOK_KEY_BASE + key.
- VAL tokens: TOK_VAL_BASE + val.
- FILLER tokens: TOK_FILLER_BASE + random filler.
- QUERY token: TOK_QUERY.

Invariant:
- The desired durable binding is previous KEY token -> current VAL token for the initial pair section.
- Filler transitions should not overwrite or diffusely contaminate finite associative banks.

Open trace tasks:
- Document exact token constants and ranges after inspecting definitions.
- Trace whether any marker semantics are available to GPU shaders or only to benchmark code.

5. Forward associative read
---------------------------
File/shader: aideen-block/src/shaders/deq_slot_attn_unified_clean.wgsl
Buffers:
- AssocBuf: durable bank memory, per slot and bank.
- AssocReadBuf: selected/read context diagnostics or read state.
- AllWeights: W_q_assoc, alpha_assoc, slot anchors, etc.

Conceptual equation:
- Query is built from previous associative source source_{t-1}.
- Banks contain key/value/usage.
- Read selects bank(s) by similarity between query and bank key.
- Read context contributes to token representation/logit path depending active flags.

Important diagnostics:
- qmax, qctx, usage, spread, bank_cos, chosen_read_prob, target logit/prob.

Observed failure:
- Read often chooses occupied banks with nontrivial probability, but bank value/logit remains near random because bank contents are diffuse.

Open trace tasks:
- Trace exact read contribution into H/Scratch and LM output.
- Determine whether LM head receives Assoc value directly, via h*, via late fuse, or diagnostic-only route for each flag.

6. Forward associative write
----------------------------
File/shader: aideen-block/src/shaders/deq_slot_attn_unified_clean.wgsl
Buffers:
- PrevHStarBuf: previous token literal source.
- AssocBuf: bank key/value/usage.
- AssocHist: pre-write bank snapshot plus metadata for backward.
- AllWeights: W_k_assoc, W_v_assoc, W_event_assoc, b_event_assoc, slot anchors.

Current conceptual equation:
- event_gate = sigmoid(W_event * feature(source_{t-1}, source_t) + b_event) when ENABLE_ASSOC_EVENT_GATE.
- bind_gate = event_gate * has_prev_hstar.
- slot allocator gives p_slot_i.
- bank allocator gives p_bank.
- allow_write = event_gate * p_slot_i * p_bank.
- effective_bind_gate = bind_gate * allow_write * transition_gate.
- effective_write_mass = effective_bind_gate^2 unless linear write flag is active.
- bank_key <- (1 - mass) bank_key + mass * key(source_{t-1}).
- bank_value <- (1 - mass) bank_value + mass * value(source_t).
- usage updates with write mass.

Known failure:
- Long gaps still produce writes across filler tokens; finite banks become an EMA/mix of many tokens.
- w_n remains proportional to sequence length in diagnostics.

Known structural fix pending validation:
- Forward now receives ENABLE_ASSOC_EVENT_GATE and related constants, matching backward.

Open trace tasks:
- Validate that event_b changes now reduce forward w_allow/w_gate.
- If not, trace event feature, parameter offsets, and AllWeights indexing.

AssocHist segmented coordinate invariant
----------------------------------------
Forward file: aideen-block/src/shaders/deq_slot_attn_unified_clean.wgsl
Backward file: aideen-backbone/src/shaders/fused_fpm_retain_bwd.wgsl
Buffer owner: RustDeqBridge.assoc_hist_buf, allocated as [batch][max_seq_len][h_slots][assoc_hist_slot_stride].
Forward token coordinates: local segment loop variable `t`, full-sequence token `global_t = shape.token_start + t`, full-sequence length `shape.seq_len`.
Backward token coordinates: `t` over `params.seq_len`, absolute index `t_abs = b * params.seq_len + t`.
Required invariant: every AssocHist write for a forward token must use the full-sequence coordinate `(batch_idx * shape.seq_len + global_t)`, because backward reads `(b * params.seq_len + t)`.
Failure mode found: local segment indexing can overwrite early history positions and leave later positions stale when internal segmented forward is active.


Stabilized Assoc multi-binding path update
------------------------------------------
- Assoc address path now treats `W_k_assoc`/`W_q_assoc` as an optional learned residual over the direct normalized token/source address. `AIDEEN_ASSOC_ADDR_LR_MULT` defaults to 0.0, preserving the literal key/query manifold during current explicit-memory training.
- When `ENABLE_ASSOC_SLOT_OWNER=false`, Assoc slots are replicated lanes, not competitors. Each active Assoc slot writes full local binding mass. If slot ownership is enabled later, `p_slot_i` returns to normalized owner probability.
- Bank allocation now uses empty-first placement for novel bindings: if a free bank exists and no existing key match is strong enough, the new binding is committed to the free bank instead of being dropped by occupied-bank protection.
- Assoc retrieval is winner-take-all by default. Soft bank mixtures are experimental because they blend multiple literal values and contaminate exact key->value recall. FPM remains the graded/contextual memory path.
- Validated default AR corridor: pairs_per_seq=8, banks=8, gaps 0/64/128/256 solved with losses around 0.0002-0.0004.

Assoc scalable capacity note - adaptive hash replication
-------------------------------------------------------
Current experimental capacity path:
- `AIDEEN_ASSOC_HASH_LANE=1` makes Assoc lane ownership deterministic rather than fully competitive.
- `AIDEEN_ASSOC_HASH_REPLICA=1` can add a deterministic adjacent lane as a second probe, but only when `ASSOC_BANKS >= 16` in the shader path.
- For `ASSOC_BANKS < 16`, replica is disabled even if the env flag is set, preserving the single-owner small-bank circuit.

Circuit invariant:
- Write and read must share the same owner/probe lane set for a binding.
- Replication is a collision-reduction mechanism, not a replacement for salience or durable slot policy.
- Literal Assoc recall remains separate from FPM contextual consolidation: Assoc stores explicit key->value evidence; FPM may summarize/contextualize but should not be required to recover literal pairs that Assoc failed to preserve.

Validated observations:
- p16/b8 is protected by the adaptive guard and remains in the working small-bank corridor.
- p24/b16 can work with replica, proving that the previous p24 failure was largely capacity/collision/routing related.
- p24/b16 variance shows the next unresolved circuit segment is write salience/replacement under lane pressure.

Capacity edge observation
-------------------------
- h8/b16 with adaptive two-lane Assoc replication solved p32 (`eval_loss=0.0941`, random=4.4067).
- This means the current token path can preserve many literal bindings when lane/bank geometry is sufficient.
- The unresolved problem is now narrower: behavior when requested bindings exceed effective lane/bank capacity, or when salience should decide which bindings persist across longer documents/segments.
